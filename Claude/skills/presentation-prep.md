# Skill: Presentation Prep

Covers slide decks, speaker notes, talk scripts, and high-level framing for mixed or technical audiences.

---

## Format

- **Slide outlines**: Structured markdown. Each slide gets a title and 2–4 key points (not full sentences — compressed logic). Visual suggestions where relevant.
- **Speaker notes**: Concise prose that tracks the argument, not a script to read verbatim. Notes should capture the *reasoning* behind each slide, not just restate the bullet points.
- **Talk scripts** (when requested): Written to sound spoken — direct, paced, and structured around transitions between ideas rather than around slides.

## Structural Principles

1. **Lead with the problem, not the method.** The audience needs to care about the question before they care about the approach.
2. **One idea per slide.** If a slide needs two ideas, it's two slides.
3. **Build the argument, don't list results.** Slides should follow a logical arc: problem → gap → approach → key result → what it means. Not: background → methods → results → conclusions as a flat list.
4. **Calibrate depth to audience.** For mixed audiences (e.g., DLT, cross-disciplinary talks), lead with intuition and framing; put technical detail in backup slides. For specialist audiences, go deeper on method and validation.
5. **End with what matters, not what's next.** The final slide should land the main takeaway, not a generic "future work" list.

## Voice Rules

- Same core voice: direct, precise, no filler.
- In presentations, slightly more accessible than in papers — shorter sentences, clearer transitions, more explicit signposting ("The key question is…", "This tells us…").
- Never hype. Let the work speak. If the result is strong, state it clearly and move on.
- Avoid: "exciting results," "we were surprised to find," or any framing that substitutes enthusiasm for substance.

## Process

1. Start by defining the talk's one-sentence takeaway. Every slide should serve that takeaway.
2. Draft the slide outline first (titles + key points). Get the logic right before adding detail.
3. Add speaker notes after the structure is locked.
4. Flag where a figure, diagram, or visualization would carry more weight than text.
5. For time-constrained talks: cut from the middle (background and methods), not from the problem or the punchline.

## Quality Checklist

- [ ] The one-sentence takeaway is clear and stated early
- [ ] The logical arc builds — it doesn't just list
- [ ] Depth is calibrated to the audience
- [ ] Each slide has one clear idea
- [ ] Speaker notes capture reasoning, not just restatements
- [ ] No hype language or empty emphasis
- [ ] Figures/visuals are flagged where they'd help
- [ ] The final slide lands the main point
- [ ] For time-constrained versions: the problem and punchline survive; the middle is compressed
